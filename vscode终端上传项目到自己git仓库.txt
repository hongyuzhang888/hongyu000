VS Code 终端把项目上传到自己的 GitHub/Git 仓库 完整教程
我给你整理了最简单、一步一步照着做的版本，全程在 VS Code 终端里操作，不用额外软件。
你电脑已经安装了 Git（没装的话先装：https://git-scm.com/）
你有自己的 GitHub/Gitee/GitLab 仓库（空仓库就行）
打开 VS Code，打开你的项目文件夹
一、打开 VS Code 终端
VS Code 顶部菜单 →终端 → 新建终端就会弹出底部的命令行窗口。
二、逐行执行以下命令（复制粘贴即可）
1. 初始化 Git（第一次上传必须执行）
git init
2. 把所有文件加入暂存
git add .
3. 提交代码
git commit -m "第一次提交项目"
4. 关联你的远程仓库
把下面的 你的仓库地址 换成你自己的仓库链接（GitHub 上复制的 HTTPS 地址）
git remote add origin 你的仓库地址示例：
git remote add origin https://github.com/你的用户名/你的仓库名.git
git remote add origin https://github.com/hongyuzhang888/hongyu000.git
5. 推送到远程仓库
git push -u origin master
如果这行报错（大概率不会），就用这行强制推送：bash运行git push -u origin master -f
三、以后修改代码后再次上传
以后改完代码，只需要 3 行：
git add .
git commit -m "更新了xxx"
git push
四、常见问题
1. 提示登录 GitHub
输入你的 GitHub 账号密码即可。
2. 提示 remote origin already exists
git remote remove origin
再重新执行关联命令。
3. 推送失败：先拉取再推送
git pull origin main --allow-unrelated-histories
git push origin main
总结

git commit -m "second commit"
git push -u origin master

fatal: 'origin' does not appear to be a git repository
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.
这个 Git 错误非常常见，核心原因只有一个：你的本地项目没有配置好名为 origin 的远程仓库地址，Git 不知道要把代码推送到哪里。
我给你整理了最简单直接的解决步骤，复制命令就能搞定：
第一步：先检查当前有没有远程仓库
在项目文件夹里打开终端，运行：
git remote -v
如果什么都不输出 → 完全没配置远程仓库
如果输出了错误的地址 → 需要重新配置
第二步：添加正确的远程仓库地址
情况 1：你用 GitHub / Gitee / GitLab
去你的代码托管平台，复制仓库的 HTTPS 地址 或 SSH 地址。
然后运行（把链接换成你自己的）：
git remote add origin https://github.com/你的用户名/你的仓库名.git
情况 2：你想删除旧的错误地址再重新添加
git remote remove origin
git remote add origin 你的仓库地址
第三步：验证是否配置成功
git remote -v
正常会显示：
plaintext
origin  https://github.com/xxx/xxx.git (fetch)
origin  https://github.com/xxx/xxx.git (push)
第四步：现在就可以正常推送了
git push -u origin main
（如果你的主分支是 master，就把 main 换成 master）
最常见的 3 个原因
刚初始化本地仓库，还没绑定远程仓库
远程仓库地址写错 / 拼写错误
克隆仓库时路径不对，或文件夹被移动过
总结
用 git remote -v 检查是否有远程
用 git remote add origin 仓库地址 绑定
再执行 git push 即可
